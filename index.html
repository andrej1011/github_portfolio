CS340 Projects
Python Networks TCP/IP MD5

Implementation of the full network stack from scratch as part of CS340 "Introduction to Computer Networking" curriculum.

Self-studied CS340: "Introduction to Computer Networking" course from Northwestern University by professor Steve Tarzia, utilizing the full lecture series available via YouTube. This in-depth course uses a bottom-up approach to teach the 7-layer model OSI model protocols as well as the application and usage of modern network protocols. Coursework culminated in the development of the following student projects:

Project 1: HTTP Server & Client: Developed a multi-threaded web server and client that manually parses raw byte streams into HTTP/1.0 messages. The client handles 301/302 redirects, content-type validation, and status code logic completely manually.

Project 2: Reliable Transport Protocol: Designed and implemented a reliable transport protocol, described as "primitive version of TCP" by the professor, over unreliable UDP (`LossyUDP`) to guarantee delivery in a simulated high-latency, packet-dropping network. Engineered mechanisms for Stop-and-Wait ARQ, MD5 checksumsfor data integrity, and a threaded listener to handle asynchronous ACKs. This project simulated kernel-level networking challenges, teaching me how reliability, flow control, and connection teardown (FIN/ACK) actually work.

Subway2D
Unity C# Game Design

A 2D endless runner game developed in Unity.

Subway2D is an endless runner game developed in Unity, created as the practical component for the university course Seminar C: Unity Game Development. Key technical contributions include a relative velocity system for simulating movement without player displacement, object pooling for memory-efficient procedural generation, and linear interpolationfor dynamic difficulty scaling. The study demonstrates that 3D gameplay loops can be preserved in 2D through precise lane-snapping logic and speed manipulation.

PONG for ZX Spectrum
Assembly Z80 ZX Spectrum

Two-player implementation of the classic arcade game PONG, built in Assembly for the ZX Spectrum.

Developed as the final project for the Computer Science Workshop at Petnica Science Center, this project demanded a deep dive into bare-metal programming. Working within the severe memory and processing constraints of the Z80 hardware, I engineered the entire game loop, physics engine, and rendering routines directly in Z80 assembly. This required manual memory management and direct hardware manipulation to achieve optimal performance, demonstrating a fundamental understanding of low-level system architecture.
